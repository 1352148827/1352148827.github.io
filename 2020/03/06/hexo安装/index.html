hexo安装

官网: https://hexo.io/zh-cn/

    Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
    Hexo可以一键部署到github上

1 安装hexo(github+hexo)

1.1 安装前提

需要安装软件
    nodejs 版本不低于8.10,建议用10.0版本以上
    git

1.2 安装步骤

// 全局安装hexo
npm install hexo-cli -g
// 创建"blog"博客项目
hexo init blog
// 进入博客项目
cd blog
// 安装依赖
npm install
// 安装git部署插件
cnpm install --save hexo-deployer-git
// 运行系统
hexo server

1.3 github创建仓库

在github上创建仓库 仓库名固定为: xxx.github.io  (xxx为github的用户名)
                        示例: 1352148827.github.io

1.4 配置_config.yml

在_config.yml中配置git的地址

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: https://github.com/1352148827/1352148827.github.io.git
  branch: master

1.5 部署到github上

在博客项目下执行命令:
    hexo deploy

1.4 hexo目录结构

hexo目录结构主义分为三个部分,一个配置文件, _config.yml是网站的配置,scaffolds目录是存放模板的,source目录主要存放资源文件的,themes目录主要是存放主题的
.
├── _config.yml        // 网站配置信息;参考:
├── package.json
├── scaffolds        // 模板文件夹    参考: https://hexo.io/zh-cn/docs/writing
├── source            // 资源文件夹是存放用户资源的地方除
|   ├── _drafts
|   └── _posts
└── themes            // 主题文件夹,Hexo 会根据主题来生成静态页面。 
                    // 参考:https://hexo.io/zh-cn/docs/themes
source目录:  _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

2 hexo使用

2.0 hexo上传github命令

hexo clean
hexo g
hexo d

2.1 hexo创建文章

hexo new 模板名 "文章名称"
hexo new post  "My New Post"

2.2 启动项目

hexo server
npm run server

2.3 编译项目

hexo generate
npm run build

2.4 清理项目

hexo clean
npm run clean

2.5 部署github

hexo deploy
npm run deploy

2.6 查看博客

https://github用户名.github.io
示例 https://1352148827.github.io
